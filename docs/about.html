🌊 Overview

Northern Beaches Surf Check is an automated surf reporting system for Sydney's Northern Beaches. It runs daily at 5:45 AM via a macOS launch agent, fetches live marine and meteorological data, processes it into a detailed surf report, and deploys it as a static webpage via GitHub Pages.

This page explains how the surf report is generated — the data sources, calculations, and decisions behind every rating and recommendation.

📡 Data Sources

Open-Meteo Marine API

Provides hourly wave and wind forecasts for the Sydney coast (-33.78, 151.30):

  • Wave Height — Significant wave height in metres (H₀)
  • Wave Period — Peak wave period in seconds (T)
  • Wave Direction — Direction waves are coming from, in degrees from north
  • Wind Speed & Direction — 10m wind speed and direction

Open-Meteo provides free, no-registration-required access to global weather models.

MHL (Manly Hydraulics Laboratory)

Historically used for real-time tide observations from Station 213470 (Sydney / Port Jackson at HMAS Penguin). However, the MHL API endpoint is currently unavailable, so the system uses a harmonic tide model instead.

🌊 Wave Height at Breaking

Rather than simply displaying the offshore swell height, we estimate how high waves will be when they break at each beach, using the breaker index formula commonly used by oceanographers:

H ≈ 0.44 · H₀ · (T / √H₀)^(2/5)

Where:

  • H = Wave height at breaking (metres)
  • H₀ = Offshore deep-water swell height (metres)
  • T = Swell period (seconds)

This formula accounts for the fact that waves grow as they move into shallower water, with longer-period swells amplifying more than shorter-period swells. A 1m, 14s swell will break at about 1.4m, while a 1m, 8s swell will only reach about 1.0m.

Directional Exposure

Not all swell reaches every beach equally. Each beach has a known aspect — the direction it faces. The offshore swell energy is scaled by a directional exposure factor:

exposure = cos(|swell_direction − beach_aspect|)

A beach facing directly into the swell (Δθ = 0°) gets 100% exposure. A beach at 90° to the swell gets 0%. The directional factor is applied to the offshore height before the breaker formula.

🌊 Tide Height Calculation

Since the MHL tide API is unavailable, we use a harmonic tide model based on the two principal tidal constituents for Sydney Harbour:

tide(t) = MSL + A_M2 · sin(2π · t/T_M2 − φ_M2) + A_S2 · sin(2π · t/T_S2 − φ_S2)
ParameterValueDescription
MSL0.9 mMean Sea Level (relative to Chart Datum)
A_M20.57 mPrincipal lunar semi-diurnal amplitude
T_M212.42 hLunar tidal period (12h 25m)
A_S20.16 mPrincipal solar semi-diurnal amplitude

The model uses an epoch-based calculation (since Jan 1, 2000) so the lunar tide advances ~50 minutes each day, matching real-world behaviour. Phase angles are calibrated for Sydney Harbour (Port Jackson / Fort Denison).

Tide Trend

The trend (rising/falling) is determined by comparing the tide height now to predictions one hour before and after the current time. Low tide heights are displayed as negative values.

⭐ Surf Rating Calculation

The surf rating (0–5 stars) combines three normalized factors:

1. Wave Height Factor (0–2.5 points)

Linear up to 2.0m: min(height / 2.0, 1.0) × 2.5

2. Period Factor (0–2.5 points)

Linear from 6s (0 points) to 16s (2.5 points): max(0, min((T − 6) / 10, 1)) × 2.5

3. Tide Factor (modifier)

Optimal tide range is 0.5–1.5m (factor = 1.0). Outside this range the factor decreases linearly to 0.5 at 0m and 3.0m.

The final rating is the product of all three factors, rounded to the nearest half star.

🏄 Board & Wetsuit Recommendations

Board Selection

Based on wave height at breaking, with adjustment for short-period swell:

  • Log — < 0.4m
  • Funboard — 0.4–0.9m
  • Shortboard — 0.9–1.5m
  • Gun — 1.5m+

If the swell period is less than 8 seconds, the recommendation is adjusted down one step (e.g. Shortboard → Funboard).

Wetsuit Guide

Based on month-derived sea-surface temperature for Sydney and the Quiksilver wetsuit guide:

  • Dec–Feb (Summer, 20–24°C): Boardshorts or rash vest
  • Mar, Nov (18–20°C): Spring suit (2mm)
  • Apr–May (16–18°C): 3/2 full wetsuit
  • Jun–Aug (Winter, 14–16°C): 4/3 full wetsuit
  • Sep–Oct (16–19°C): 3/2 full wetsuit

🤖 Automation & Deployment

The entire system runs automatically with no manual intervention:

  1. 5:45 AM daily — A macOS launch agent triggers surf_report.py
  2. Python script fetches live data from Open-Meteo APIs
  3. Calculates beach-specific conditions, ratings, and recommendations
  4. Generates a static HTML page (docs/index.html)
  5. Commits the changes to the local git repository
  6. Pushes to GitHub, which triggers GitHub Pages rebuild
  7. The live site is updated within minutes at https://makrmark.github.io/surforecast/

All code is open source and available at github.com/makrmark/surfcheck.

⚠️ Limitations & Notes

  • Tide data is from a harmonic model, not real-time observations. Accuracy is approximately ±0.2m under normal conditions.
  • The breaker formula is a generalized rule-of-thumb; actual breaking wave heights depend on local bathymetry, reef structure, and sandbar positions.
  • Wind data is for the offshore Sydney location, not beach-specific local wind conditions.
  • Wetsuit recommendations are based on monthly climatological averages, not real-time sea temperature.
