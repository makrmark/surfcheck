🌊 Overview

Northern Beaches Surf Check is an automated surf reporting system for Sydney's Northern Beaches. It runs daily at 5:45 AM via a macOS launch agent, fetches live marine and meteorological data, processes it into a detailed surf report, and deploys it as a static webpage via GitHub Pages.

This page explains how the surf report is generated — the data sources, calculations, and decisions behind every rating and recommendation.

📡 Data Sources

Open-Meteo Marine API

Provides hourly wave and wind forecasts for the Sydney coast (-33.78, 151.30):

  • Wave Height — Significant wave height in metres (H₀)
  • Wave Period — Peak wave period in seconds (T)
  • Wave Direction — Direction waves are coming from, in degrees from north
  • Wind Speed & Direction — 10m wind speed and direction in km/h and knots

Open-Meteo provides free, no-registration-required access to global weather models.

Tide Data — Harmonic Tide Model

Rather than relying on an external tide API (the MHL endpoint is currently unavailable), the system uses a harmonic tide model calibrated for Sydney Harbour. See the Tide section below for full details.

🌊 Wave Height at Breaking

Rather than simply displaying the offshore swell height, we estimate how high waves will be when they break at each beach, using the breaker index formula commonly used by oceanographers:

H ≈ 0.44 · H₀ · (T / √H₀)2/5

Where:

  • H = Wave height at breaking (metres)
  • H₀ = Offshore deep-water swell height after directional exposure (metres)
  • T = Swell period (seconds)

A 1m, 14s swell will break at about 1.4m, while a 1m, 8s swell only reaches about 1.0m. Longer-period swells amplify more.

Directional Exposure

Each beach has a known aspect — the direction it faces. The offshore swell energy is scaled by a directional exposure factor:

exposure = cos(|swell_direction − beach_aspect|)

A beach facing directly into the swell (Δθ = 0°) gets 100% exposure. A beach at 90° to the swell gets 0%. The directional factor is applied to the offshore height before the breaker formula.

🌊 Tide Height Calculation

Using a harmonic tide model based on the two principal tidal constituents for Sydney Harbour (Port Jackson / Fort Denison):

tide(t) = MSL + AM2·sin(2π·t/TM2−φM2) + AS2·sin(2π·t/TS2−φS2)
ParameterValueDescription
MSL0.9 mMean Sea Level (relative to Chart Datum)
AM20.57 mPrincipal lunar semi-diurnal amplitude
TM212.42 hLunar tidal period (12h 25m)
AS20.16 mPrincipal solar semi-diurnal amplitude
TS212.00 hSolar tidal period (exactly 12h)

The model uses an epoch-based calculation (since Jan 1, 2000) so the lunar tide advances ~50 minutes each day. Phase angles are calibrated for Sydney Harbour.

Tide Trend

The trend (rising/falling/high/low/slack) is determined by comparing the tide height now to predictions one hour before and after. Low tide heights are displayed as negative values with a trend emoji (📈, 📉, 🔺, 🔻, ➖).

🌊 Wind & Tide Effects on Surf Quality

The best beach selection goes beyond raw wave height — it considers how wind and tide conditions shape wave quality.

Wind Quality — Offshore vs Onshore

The angle between the wind direction and the swell direction determines wave quality:

  • Offshore wind (wind opposite to waves, ≈180° apart) — clean, well-shaped waves. Score boosted.
  • Onshore wind (wind in the same direction as waves) — choppy, messy waves. Score reduced.
  • Cross-shore wind (wind perpendicular to waves) — neutral.

The wind quality factor ranges from 0.0 (onshore) to 1.0 (offshore) and adjusts the composite score by up to 25%.

Tide Stage Quality

Where the tide is in its cycle significantly affects how waves break:

  • 📈 Rising tide (incoming) — Acts as a natural "push" that organises the swell and improves wave consistency. Best time for surfing — bonus applied.
  • 📉 Falling tide (outgoing) — Can generate strong rip currents, sometimes degrading wave shape. Slight penalty.
  • 🔺 High tide — More water over sandbars; waves can become soft, fat, and slow. If too high, waves may "drown out" entirely. Penalty.
  • 🔻 Low tide — Shallow water, steeper/hollower waves. Great for advanced surfers but harder paddle-out. Slight penalty for danger.
  • ➖ Slack tide — Neutral.

The tide stage quality factor adjusts the composite score by up to 25%.

composite = precise_rating × (0.5 + 0.25 × wind_quality + 0.25 × tide_quality)

⭐ Surf Rating & Best Beaches

Star Rating (0–5 ★)

The surf rating combines three normalized factors, then rounds to the nearest half star for display:

  • Wave Height Factor (0–2.5 pts): min(height/2.0, 1.0) × 2.5
  • Period Factor (0–2.5 pts): max(0, min((T-6)/10, 1)) × 2.5
  • Tide Height Factor (modifier): 1.0 at optimal tide (0.5–1.5m), decreases linearly outside that range

The precise (unrounded) rating is used internally for rankings — so a beach with 2.05 ranks above one with 1.89, even though both display as ★★☆☆☆.

🏆 Best Beaches Selection

The top 3 beaches are selected using a composite score that combines the precise surf rating with wind and tide quality:

composite = precise_rating × (0.5 + 0.25 × wind_quality + 0.25 × tide_quality)

This rewards beaches with good wave height and favourable conditions (offshore wind, rising tide) while penalising those with onshore wind or poor tide stage.

🏄 Board & Wetsuit Recommendations

Board Selection

Based on wave height at breaking, with adjustments for swell period:

  • Longboard / Log — < 0.3m
  • Longboard / Funboard / Groveller — 0.3–0.6m
  • Fish / Funboard / Mid-Length — 0.6–1.0m
  • Shortboard / Fish / Mid-Length — 1.0–1.5m
  • Shortboard / Step-Up — 1.5–2.0m
  • Step-Up — 2.0m+

Short period (< 8s): Favour higher-volume boards (e.g., Shortboard → Fish/Funboard).

Long period (>= 12s) with size (>= 1.2m): Favour a Step-Up for powerful ground swell.

Wetsuit Guide

Based on month-derived sea-surface temperature for Sydney and the Quiksilver wetsuit guide:

  • Dec–Feb (Summer, 20–24°C): Boardshorts or rash vest
  • Mar, Nov (18–20°C): Spring suit (2mm)
  • Apr–May (16–18°C): 3/2 full wetsuit
  • Jun–Aug (Winter, 14–16°C): 4/3 full wetsuit
  • Sep–Oct (16–19°C): 3/2 full wetsuit

🤖 Automation & Deployment

The entire system runs automatically with no manual intervention:

  1. 5:45 AM daily — macOS launch agent triggers surf_report.py
  2. Python script fetches live data from Open-Meteo APIs
  3. Calculates beach-specific conditions, ratings, and recommendations
  4. Generates a static HTML page (docs/index.html)
  5. Commits changes to the local git repository
  6. Pushes to GitHub, which triggers automatic GitHub Pages rebuild
  7. Live site updates within minutes

All code is open source at github.com/makrmark/surfcheck.

⚠️ Limitations & Notes

  • Tide data is from a harmonic model, not real-time observations. Accuracy is approximately ±0.2m.
  • The breaker formula is a generalised rule-of-thumb; actual breaking heights depend on local bathymetry, reef structure, and sandbars.
  • Wind data is for the offshore Sydney location, not beach-specific local conditions.
  • Wetsuit recommendations are based on monthly climatological averages, not live sea temperature.
